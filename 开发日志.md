# 天爱星Agent 开发全记录：从零构建一个 Android AI 助手

> 这是一篇面向技术博客的完整开发日志，记录了 9 次迭代打包中每一个技术决策、架构演化和踩坑修复过程。
> 项目最终交付物为一个 102 MB 的 APK，包含多模型切换、工具调用、Room 持久化、AlarmManager 提醒、43 张可切换背景等完整功能。

---

## 1. 项目起点：我们要构建什么

**天爱星Agent** 是一个运行在 Android 上的智能助手，能接入多种大语言模型（DeepSeek / OpenAI / 通义千问 / 自部署），调用系统工具（打开应用、记事、天气、提醒），具备持久化记忆和多会话管理能力。架构上预留了 Live2D 虚拟角色和 TTS 语音合成接口。

| 层级 | 技术 | 理由 |
|------|------|------|
| UI | Jetpack Compose + Material3 | 声明式 UI，协程原生 |
| 架构 | AndroidViewModel + StateFlow | 生命周期安全，响应式 |
| 数据库 | Room (SQLite) | 官方 ORM，4 张表 |
| HTTP | OkHttp 4.12 | 轻量，OpenAI 兼容 API |
| 构建 | AGP 8.2.2 + Kotlin 1.9.22 + kapt | Groovy Gradle |
| 最低/目标 | API 26 / 34 | 覆盖 95%+ 设备 |

## 5. 打包 #4 — PlannerAgent 重写

**从 JSON 到文本标记**:

```
TOOL: add_reminder
PARAMS: {"title": "开会", "datetime": "2025-06-19 15:00"}
```

LLM 更容易输出纯文本标记，解析逻辑简化：

```kotlin
repeat(3) {  // 最多 3 轮工具调用
    val toolName = response.lines().find { it.startsWith("TOOL:") }
        ?.removePrefix("TOOL:")?.trim() ?: return response
    val params = JSONObject(response.lines().find { it.startsWith("PARAMS:") }
        ?.removePrefix("PARAMS:")?.trim() ?: return response)
    val result = tools.find { it.name == toolName }?.execute(params) ?: return response
    messages.add("tool" to "Tool result: $result")
    response = llmClient.chat(messages)
}
```

**LLMClient 单例**: `getInstance(context)` — 全局唯一 OkHttp 连接池。

**ChatScreen Material3 化**: `ModalNavigationDrawer` + `Scaffold` + `Surface` 气泡 + `OutlinedTextField` 输入栏。旧组件（CharacterPanel 等）转为预留。

**删除 6 个文件**: AgentAction, PromptBuilder, ToolCall, Message (model), ShortTermMemory, ToolRegistry。

## 7. 打包 #6 — Live2D/TTS 接口 + 背景系统 (~102 MB)

### 接口预留（NoOp 模式）

```kotlin
interface Live2DController {
    val isReady: Boolean
    fun loadModel(modelPath: String)
    fun playMotion(group: String, index: Int, priority: Int)
    fun setExpression(expressionId: String)
    fun setLipSync(volume: Float)        // ← TTS 联动
    fun setEyeFollow(x: Float, y: Float)
    fun release()
}

interface TTSController {
    val isSpeaking: Boolean
    fun speak(text: String, onPlaybackProgress: (Float) -> Unit = {}, ...)
    fun stop(); fun setRate(rate: Float); fun setPitch(pitch: Float)
    fun release()
}
```

**联动链路**: AI 回复 → `TTSController.speak(onPlaybackProgress = { volume → live2DController.setLipSync(volume) })` → 音频播放 → 实时口型同步。

### 背景系统

```
BackgroundSource (sealed)
├── Asset("bg_default_1.png")   ← 43 张预置图
├── Custom(uri)                 ← 相册选择 + persistableUriPermission
└── SolidColor                  ← 兜底

BackgroundViewModel: SharedPreferences 持久化 + randomBackground() + setCustomBackground()
ChatBackground: Asset→BitmapFactory / Custom→Coil AsyncImage / SolidColor→纯色
BackgroundSettingsSheet: ModalBottomSheet 三入口（随机/相册/恢复）
```

Coil: `io.coil-kt:coil-compose:2.6.0`。43 张 SD 生成图 → `assets/backgrounds/`。

## 8. 打包 #7 — ChatScreen v3 图层叠加

### 核心问题

ChatScreen v2 的 Scaffold 不透明 → 背景图被遮挡。

### 解决方案：三层叠加 + 半透明

```
Box(fillMaxSize)
├── ChatBackground(background)          ← 底层：背景图
├── Live2DView (占位)                   ← 中层：角色
└── Scaffold(Color.Transparent)         ← 顶层：半透明 UI
    ├── TopAppBar(alpha=0.85) + MoreVert → BackgroundSettingsSheet
    ├── Surface 气泡(alpha=0.92)
    └── ChatInputBar(alpha=0.90)
```

**Manifest 补全**: `android:icon` + `android:roundIcon`。

**TTS 自动朗读**:

```kotlin
LaunchedEffect(messages) {
    messages.lastOrNull()?.let {
        if (it.role == "assistant")
            ttsController.speak(it.content) { volume → live2D.setLipSync(volume) }
    }
}
```

data class LLMConfig(
    val provider: LLMProvider = DEEPSEEK,
    val apiKey: String = "",
    val model: String = "deepseek-chat",
    val customUrl: String = "",    // CUSTOM 时生效
    val temperature: Double = 0.7,

### LLMConfigStore（SharedPreferences 持久化）

```kotlin
object LLMConfigStore {
    fun save(context: Context, config: LLMConfig)   // 保存到 SharedPreferences
    fun load(context: Context): LLMConfig            // 读取并反序列化
}
```

### LLMClient 动态更新

```kotlin
fun updateConfig(newConfig: LLMConfig) { config = newConfig }

companion object {
    fun applyConfig(context: Context, newConfig: LLMConfig) {
        LLMConfigStore.save(context, newConfig)  // 持久化
        INSTANCE?.updateConfig(newConfig)         // 运行时立即生效
    }
}
```
